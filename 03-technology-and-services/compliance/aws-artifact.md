# AWS Artifact

## Service Overview
AWS Artifact is a self-service portal that provides on-demand access to AWS compliance reports, security and compliance documentation, and agreements needed for regulatory audits and certifications.

**Simple Analogy:** Think of AWS Artifact as a secure digital filing cabinet in a compliance office where you can instantly access all of AWS's official certificates, audit reports, and legal agreements - like having a librarian who immediately hands you any compliance document you need for your auditors, without waiting weeks for paperwork to arrive by mail.

## Foundational Concepts

### What Problem Does AWS Artifact Solve?
In traditional IT environments, businesses struggle with:
- **Lengthy compliance processes** - Waiting weeks or months to receive compliance documentation from vendors
- **Audit preparation delays** - Difficulty gathering necessary security certifications and reports
- **Regulatory complexity** - Navigating multiple compliance frameworks (HIPAA, PCI DSS, SOC, ISO)
- **Documentation management** - Keeping track of updated compliance reports and certifications

### Core Value Proposition
AWS Artifact provides **instant access to compliance documentation** by:
- **Eliminating wait times** - Download compliance reports immediately, 24/7
- **Centralizing documentation** - All AWS compliance materials in one secure location
- **Simplifying audits** - Provide auditors with official AWS security documentation instantly
- **Maintaining currency** - Access to the most up-to-date compliance reports and certifications

### Key Components ⭐ **EXAM FOCUS**

**1. AWS Artifact Reports**
- Compliance reports from third-party auditors
- Security and compliance documentation
- Examples: SOC reports, PCI DSS attestations, ISO certifications

**2. AWS Artifact Agreements**
- Legal agreements between AWS and customers
- Business Associate Addendum (BAA) for HIPAA compliance
- Non-Disclosure Agreements (NDAs)

**3. Self-Service Access**
- No AWS support ticket required
- Immediate download capability
- Available through AWS Management Console

## Security and Compliance (CLF-C02 Focus) ⭐ **EXAM FOCUS**

### AWS Shared Responsibility Model

**AWS Responsibilities (Security OF the Cloud):**
- **Maintaining compliance certifications** - Undergoing audits and maintaining certifications
- **Creating compliance reports** - Producing SOC, PCI DSS, ISO, and other reports
- **Infrastructure security** - Securing the AWS Artifact portal and document storage
- **Report accuracy** - Ensuring compliance documentation is accurate and current
- **Global compliance** - Maintaining certifications across all AWS regions and services

**Customer Responsibilities (Security IN the Cloud):**
- **Accessing appropriate reports** - Downloading relevant compliance documentation for their use cases
- **Reviewing documentation** - Understanding which AWS services meet their compliance needs
- **Executing agreements** - Signing necessary agreements like BAA for HIPAA compliance
- **Sharing with auditors** - Providing AWS compliance reports to their own auditors
- **Implementing controls** - Configuring AWS services to meet their specific compliance requirements

### Compliance Frameworks Supported
- **SOC (Service Organization Control)** - SOC 1, SOC 2, SOC 3 reports
- **PCI DSS** - Payment Card Industry Data Security Standard
- **ISO Certifications** - ISO 9001, ISO 27001, ISO 27017, ISO 27018
- **HIPAA** - Health Insurance Portability and Accountability Act (BAA available)
- **FedRAMP** - Federal Risk and Authorization Management Program
- **GDPR** - General Data Protection Regulation documentation
- **HITRUST** - Health Information Trust Alliance certification

### Key Security Features
- **IAM integration** - Control who can access AWS Artifact in your organization
- **Secure access** - All documents accessed through encrypted connections
- **Audit trail** - CloudTrail logs all AWS Artifact access activities
- **Agreement management** - Track which agreements have been accepted

## Common Use Cases (Scenario-Based) ⭐ **EXAM FOCUS**

### 1. Healthcare Compliance (HIPAA)
**Business Scenario:** Healthcare startup needs to demonstrate HIPAA compliance to store and process protected health information (PHI) in AWS

**AWS Artifact Solution:**
- Download AWS HIPAA compliance documentation and certifications
- Execute the Business Associate Addendum (BAA) electronically through AWS Artifact
- Provide AWS compliance reports to healthcare auditors
- Access documentation showing which AWS services are HIPAA-eligible

### 2. Payment Processing Compliance (PCI DSS)
**Business Scenario:** E-commerce company needs to prove their payment processing infrastructure meets PCI DSS requirements

**AWS Artifact Solution:**
- Access AWS PCI DSS Attestation of Compliance (AOC)
- Download PCI DSS Responsibility Summary documents
- Provide documentation to payment card auditors (QSAs)
- Demonstrate that AWS infrastructure meets PCI DSS Level 1 requirements

### 3. Financial Services Audit Preparation
**Business Scenario:** Financial services firm undergoing SOC 2 audit needs to provide evidence of their cloud provider's security controls

**AWS Artifact Solution:**
- Download AWS SOC 1, SOC 2, and SOC 3 reports
- Provide reports to external auditors as evidence of AWS security controls
- Access ISO 27001 certification for additional security documentation
- Demonstrate AWS's commitment to security and compliance

### 4. International Data Protection Compliance
**Business Scenario:** Global company needs to demonstrate GDPR compliance for European customer data stored in AWS

**AWS Artifact Solution:**
- Access AWS GDPR compliance documentation and data processing agreements
- Download ISO 27018 certification for cloud privacy
- Execute AWS Data Processing Addendum (DPA) for GDPR requirements
- Provide documentation to data protection authorities and customers

## Related Core Services ⭐ **EXAM FOCUS**

### 1. AWS IAM (Identity and Access Management)
**Relationship:** IAM controls who can access AWS Artifact within your organization
- **Access control** - IAM policies determine which users can download compliance reports
- **Permission management** - Grant specific team members access to AWS Artifact
- **Audit access** - Restrict compliance documentation access to authorized personnel only
- **Role-based access** - Assign AWS Artifact permissions based on job functions

### 2. AWS CloudTrail
**Relationship:** CloudTrail logs all AWS Artifact access and agreement activities
- **Audit logging** - Records who accessed which compliance reports and when
- **Agreement tracking** - Logs when agreements like BAA are accepted
- **Compliance evidence** - Provides audit trail of compliance documentation access
- **Security monitoring** - Detect unauthorized access attempts to compliance reports

### 3. AWS Organizations
**Relationship:** Organizations can manage AWS Artifact agreements across multiple accounts
- **Centralized management** - Accept agreements once for all accounts in organization
- **Organizational agreements** - BAA and other agreements apply to all member accounts
- **Simplified compliance** - Manage compliance documentation across entire organization
- **Consolidated access** - Control AWS Artifact access at organizational level

### 4. AWS Config
**Relationship:** Config helps implement and monitor compliance controls documented in AWS Artifact
- **Compliance validation** - Verify AWS resources meet compliance requirements
- **Configuration tracking** - Ensure resources maintain compliant configurations
- **Audit support** - Provide evidence of compliance alongside AWS Artifact documentation
- **Continuous monitoring** - Monitor compliance posture referenced in AWS Artifact reports
